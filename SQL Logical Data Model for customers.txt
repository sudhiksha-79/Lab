Standard for defining tables and columns:
In the software industry there are certain naming standards followed at each level.
In the database the following conventions needs to be followed:
1. Table names must have indicators of the functional area.
2. Usually, abbreviations are used while defining the tables and columns. Standard abbreviation sample is given. One can
have their own standards but documenting them for others to understand is very important.
3. Underscores separate the distinct terms that define the Table or Column as per the standard conventions. Tablename and
column name can have multiple underscores.
